# Skills 管理

Skills 管理模块使用统一 Skills 根目录作为本地 Skill 内容的真实来源，并通过 Workbench SQLite 记录分类、全局启用、项目级启用、同步方式和冲突状态。前端入口位于 `src/App.tsx`，后端 command 入口为 `src-tauri/src/skills.rs`，类型、数据库、文件系统同步、工具目标、分类、自定义工具、导入、根目录迁移、市场和 CLI 适配位于 `src-tauri/src/skills/`。

## 当前能力

- 通过“同步 Skills”刷新统一 Skills 根目录中包含 `SKILL.md` 的 Skill，并检查已注册工具全局目录中的外部 Skills。
- 外部工具 Skills 同步采用先只读发现、再由用户按状态确认的流程；同步弹窗按 `新增 / 冲突 / 已存在 / 不可用` 分 Tab 展示候选，确认后导入统一 Skills 根目录，并接管对应工具目录目标。
- 从 ZIP 文件或已解压文件夹导入一个或多个 Skills；同名冲突项在结果弹窗中由用户选择跳过或覆盖。
- 从 public GitHub 仓库、tree 路径或 `SKILL.md` blob 链接扫描标准 Skill 候选；用户预览 `SKILL.md` 后勾选导入，导入单位固定为 `SKILL.md` 所在目录整体。
- 从 `skills.sh` 技能市场浏览、搜索、查看详情、安装和卸载 GitHub 来源 Skills。
- 在 `更新` 子视图检查和更新由 Workbench 管理的远程来源 Skills；当前支持 `skills_sh` 和可重新定位的 GitHub 分支来源。
- 支持远程来源 Skill 的单项更新、选中批量更新和更新全部可更新项。
- 更新页复选框只对状态为可更新的 Skill 启用；已是最新、检查失败或更新执行中的条目保持禁用，避免批量更新误选不可执行项。
- 市场页支持按状态筛选，并按当前已加载结果统计已加载、已安装、未安装、可更新、不支持和当前筛选结果数量。
- 技能市场无搜索时展示 `skills.sh` 首页榜单结果；搜索时调用匿名搜索接口按需增大 `limit` 加载更多结果，分页到已加载末尾时可继续触发下一批加载。
- 技能市场搜索请求至少需要 2 个字符，匿名搜索单次 `limit` 由后端限制在 2000 以内；当前未接入需要认证的官方分页 API。
- 技能市场来源列展示 GitHub `owner/repo` 来源的仓库或组织头像；非 GitHub 来源或图标加载失败时使用本地文本回退。
- 市场列表在当前应用进程内缓存，手动刷新会绕过缓存，安装完成后会清除缓存并刷新本地状态；安装任务完成后返回市场页会重新加载列表以同步已安装状态。
- 市场初次加载使用列表与详情骨架屏，加载失败时展示错误和重试入口。
- 市场安装使用后端阶段事件反馈百分比进度，同一时间只允许执行一个市场安装操作；安装过程中切换 Skills 子视图或离开 Skills 主页面再返回时，会恢复显示当前安装进度或最终结果。
- 市场状态使用语义图标标签，“已安装”和“不支持”只展示状态，不作为可点击安装按钮；已安装条目的操作列提供“卸载”动作。
- 打开统一 Skills 根目录和单个 `SKILL.md` 文件。
- 删除统一根目录中的 Skill，并清理 Workbench 管理的全局和项目级启用记录。
- 通过 `skill_categories` 管理 Workbench 内分类，Skill 元信息只保存 `category_id`。
- 为每个 Skill 维护一个 Workbench 内分类；缺失分类统一归入“未分类”。
- 在 Skills 列表中通过分类下拉选择已有分类或创建新分类。
- 在分类管理弹窗中新增、重命名、删除迁移和合并分类。
- 按名称或描述搜索 Skills。
- 按分类、启用状态、启用工具和启用项目筛选 Skills。
- 本地 Skills、技能市场和更新页支持分页浏览；筛选或搜索变化后回到第一页，翻页后详情切换到当前页可见项；技能市场搜索翻到已加载结果末尾时会先加载更多再进入目标页。
- 工具筛选同时匹配全局启用和项目级启用。
- 项目筛选只匹配在指定项目下启用过的 Skills。
- 工具和项目组合筛选只匹配在该项目下启用了该工具的 Skills。
- 在 Skills 列表中切换支持的全局工具启用状态。
- 全局工具当前包括 Codex、Claude Code、OpenCode、DevEco Code、Hermes、Kimi Code、Pi Agent、Gemini CLI、Qwen Code、Goose、Kilo Code、Cline、Roo Code、Factory Droid、Amp、Kiro CLI 和 Junie CLI。
- 设置页支持新增、编辑和删除自定义 Agent 工具；自定义工具只提供全局 Skills 目录，不支持项目级启用。
- 自定义工具图标由用户选择本地文件，保存到 `~/.workbench/tool-icons/` 后由 Workbench 管理。
- 自定义工具新增时由后端根据工具名称生成内部 key；用户界面不展示或要求填写该内部标识。
- 自定义工具名称不能与内置工具或其他自定义工具重复；表单错误在弹窗内展示。
- 设置页“支持的工具目录”展示工具图标、内置/自定义类型、路径、排序、打开目录和可用状态。
- Skills 列表中的全局工具列按用户设置顺序展示前 4 个彩色工具图标，剩余工具通过 `+N` 浮层展示和操作。
- Skills 列表隐藏可见滚动条，并通过收紧列宽和保留操作列空间减少默认窗口下的横向溢出。
- 在 Skill 详情中按项目批量启用或停用支持项目级 Skills 的工具；当前项目级工具为 Codex、Claude Code 和 OpenCode。
- 设置页支持调整全局工具展示顺序。
- 设置页切换统一 Skills 根目录前会要求确认，并记录上一个根目录用于后续迁移检查。
- 设置页支持检查旧统一根目录中的可迁移 Skills，并由用户显式迁移到当前统一根目录。
- 设置页支持检查仍指向旧统一根目录的 Workbench 受管启用目标，并由用户显式重建到当前统一根目录。
- 当检测到全局工具目录中的内容冲突时，通过 Skill 级冲突面板选择唯一版本源并统一同步。

## 数据所有权

- 统一 Skills 根目录保存 Skill 内容的唯一真实副本。
- 外部工具目录发现阶段只读执行，不创建目录、不复制文件、不写数据库。
- 从外部工具目录同步 Skill 时，用户确认的项会复制到统一 Skills 根目录，并将对应工具目录目标替换为 Workbench 管理的链接或副本；替换前会备份原工具目录内容。
- 外部同步中的新增项可以默认导入并接管；同名相同内容项不作为主操作项，本次不接管、不修改工具目录，若没有其他待处理项只通过 toast 汇总，若提交时发现已变为同名相同内容则自动跳过；同名不同内容项必须由用户选择保留 Workbench 版本或使用外部版本后才能接管。
- 外部同步提交期间会禁用重复操作并显示同步中状态；成功后关闭弹窗、刷新 Skills 列表和工具启用状态，只在失败、冲突或无效结果时保留可读原因。
- 用户跳过的外部工具目录 Skill 不进入 Workbench 管理模型。
- ZIP / 文件夹导入遇到统一 Skills 根目录中的同名 Skill 时默认不覆盖；用户在导入结果弹窗中显式选择覆盖后，后端先备份旧版本，再替换统一根目录中的内容。
- GitHub 导入只支持 public 仓库和标准 `SKILL.md` 目录；不执行仓库脚本，不支持私有仓库，不支持用户自由拼装文件或文件夹。
- GitHub 导入默认通过本机 `git clone --depth 1` 克隆 public 仓库到隔离临时目录后扫描候选；commit 固定版本链接使用 GitHub codeload ZIP 直下，不依赖 GitHub REST API；仓库根目录只有在存在 `SKILL.md` 时才作为整个仓库 Skill 候选。
- GitHub 候选预览、导入、内容 hash 和更新判断都不把 `.git` 元数据计入 Skill 内容。
- GitHub branch 或默认分支来源写入 `skill_sources` 并支持后续检查更新；tag 或 commit 固定版本可导入但不参与更新检查。
- 切换统一 Skills 根目录只更新当前真实来源并记录上一个根目录，不自动迁移旧根目录内容，不自动重建工具目录链接或副本，也不删除旧根目录。
- 旧根目录迁移只在用户显式执行时发生；同名相同内容跳过，同名不同内容返回冲突，不覆盖。
- 受管启用目标重建只处理 `skill_enablements` 中记录的 Workbench 受管目标；未受管或已被用户修改的工具目录内容不会被覆盖。
- `skills.sh` 是当前唯一接入的在线技能市场来源；市场安装、更新检查和更新执行通过隔离的临时 HOME 调用 `npx -y skills add` 提取 Skill 内容，再由 Workbench 复制到统一 Skills 根目录。
- `skills.sh` 市场浏览有榜单和搜索两种列表模式：榜单来自首页内嵌数据，搜索来自匿名 `/api/search`；两者都不是 Workbench 的全量市场目录缓存。
- 市场安装和更新依赖用户环境中的 Node.js、npm 和 npx；缺失时在市场页展示明确提示，不进入安装流程。
- `skill_sources` 记录由 Workbench 管理的远程 Skill 来源元数据，用于更新检查和更新执行；当前来源类型为 `skills_sh` 和 `github`。
- 卸载或删除由 `skills.sh` 安装的 Skill 时，同步清理对应 `skill_sources` 记录，避免市场和更新页保留幽灵来源状态。
- 非 GitHub `owner/repo` 来源的 `skills.sh` 条目暂不支持 Workbench 安装。
- 非 GitHub `owner/repo` 来源的市场条目只展示列表数据生成的本地详情，不请求远程详情接口，避免把不可安装条目的 404 当作用户可恢复错误。
- 市场安装后的 Skill 默认不启用到任何 Agent 工具目录。
- 市场详情中的安装入口只保留在列表操作列；右侧详情用于展示来源、状态、安全提示和 `SKILL.md` 预览。
- 市场安装和更新只接受单层 Skill 目录名，拒绝绝对路径、父目录和嵌套路径后再访问统一 Skills 根目录。
- Workbench 只依赖 CLI 生成的临时 `.agents/skills/<skillId>/SKILL.md` 和目录内容 hash，不解析 CLI 输出作为事实源。
- 分类只用于 Workbench 内整理，不改变 Skill 文件目录、工具目录或符号链接结构。
- `未分类` 是系统分类，后端保证存在且不允许删除或重命名。
- 删除或合并分类只迁移该分类下 Skills 的 `category_id`，不删除 Skills。
- 全局启用和项目级启用由 `skill_enablements` 记录；目标目录中的 Workbench 管理链接或副本是派生结果。
- 工具目标由后端内置注册表和 `custom_tool_targets` 自定义表合并得到；用户展示顺序保存在 `app_settings.tool_target_order`。
- 自定义工具 key 是后端内部标识，新增时自动生成，编辑名称时保持不变。
- 设置页打开不存在的工具 Skills 目录时，用户确认后会创建目录并打开。
- 不支持项目级 Skills 的工具只能用于全局启用，后端会拒绝 project scope 启用。
- 删除自定义工具只删除 Workbench 配置、展示排序和该工具相关启用记录，不删除外部工具目录。
- Workbench 停用、删除或市场卸载 Skill 时，只清理 Workbench 管理的链接或副本，不删除未被 Workbench 管理的外部工具目录内容。

## 同步边界

- 启用 Skill 时使用 Auto 同步：优先创建符号链接；无法创建符号链接时回退为 Copy。
- 外部工具目录同步接管同样使用 Auto 同步；接管前先备份原工具目录目标，再创建 Workbench 管理链接或 Copy 副本，并写入启用记录。
- 统一根目录切换后，已存在的符号链接不会自动指向新根目录；用户可通过重建受管目标删除旧受管链接并创建指向当前根目录的新链接。
- 统一根目录切换后，Copy 副本不会自动随新根目录变化；重建 Copy 副本前必须能确认目标仍是 Workbench 受管副本。
- Copy 副本不会在扫描时自动覆盖。
- 目标目录已有内容且不属于 Workbench 管理时，不自动覆盖；冲突必须由用户选择唯一版本源后解决。
- 解决冲突前，后端会备份被替换版本。
- `skills.sh` 更新检查通过重新调用官方 CLI 提取远端 Skill 并计算内容 hash 判断是否变化；该 hash 只用于本地更新检测，不是安全签名。
- GitHub 分支来源更新检查通过重新克隆同一 repo/ref/path 并计算内容 hash 判断是否变化；GitHub tag 或 commit 固定版本返回不支持更新。
- 远程来源更新执行前会备份统一根目录中的旧版本。
- 远程来源更新只替换统一 Skills 根目录内容，不自动重同步已启用的 Copy 副本。
- 批量更新按项执行并返回逐项结果，单项失败不会中断后续项。
- 批量更新目标来自用户勾选的可更新项；不可更新项不会进入批量更新目标集合。
- `SKILL.md` 预览会移除从 `skills.sh` 页面数据带入的 Next.js 内部占位标记。
