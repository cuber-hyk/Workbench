# Workbench App PRD

## 1. 产品概述

Workbench App 是一个本地优先的 AI 开发工作台，用于统一管理个人开发项目、skills 和 AI Radar 信息。

它不是目录型工作流，也不是现有工具的插件，而是一个独立桌面软件。现有项目仅作为功能参考和业务灵感来源，不合并代码，不拼成 monorepo。

## 2. 产品目标

第一阶段目标是实现一个最小可用版本，让用户可以在一个本地桌面应用中完成以下事情：

- 记录和查看本地开发项目。
- 通过启动按钮执行项目启用的启动项。
- 从统一根目录管理和分类本地 skills。
- 为全局工具或指定项目启用 skills。
- 从 ZIP 文件或已解压文件夹导入 skills。
- 维护本地 AI Radar 条目。

MVP 重点是建立清晰、可扩展但不过度设计的工作台骨架。

## 3. 用户画像

目标用户是本机长期使用 AI 辅助开发的个人开发者。

用户通常同时维护多个本地项目，并使用 Codex、Claude Code、OpenCode 和 skills 等工具。用户希望把项目入口和 skills 集中管理，而不是分散在文件夹、终端历史和多个工具配置目录中。

## 4. 产品原则

- 本地优先：数据默认保存在本机。
- 独立软件：不作为现有项目或工具的插件。
- 简单优先：第一版不做复杂插件系统、市场、同步或自动化平台。
- 单一实现：不保留多套并行实现，不为历史项目做兼容层。
- 统一来源：Workbench Skills 根目录保存 skill 的唯一真实副本，工具和项目目录通过 Auto 同步启用。
- 渐进增强：先把项目启动、skills 管理和本地信息管理做稳，再扩展更复杂的开发流程。

## 5. MVP 范围

### 5.1 项目管理

用户可以添加本地项目路径，并在 Workbench 中维护项目基本信息。

功能需求：

- 添加本地项目路径。
- 支持通过系统目录选择器填充项目路径和启动工作目录。
- 展示项目名称、路径、备注、标签。
- 支持搜索项目。
- 支持查看项目详情。
- 支持打开项目目录。
- 支持归档和恢复项目记录。
- 支持为项目配置一个或多个启动项。
- 每个启动项包含名称、启动命令、启动工作目录和启用状态。
- 支持通过启动按钮为所有启用启动项创建 Workbench 内嵌启动会话。

Workbench 的第一版启动会话为非交互式，只展示本次启动的内存日志。项目详情面板展示启动摘要和日志入口，完整输出在项目模块内的启动日志详情页中按“全部”或单个启动项查看。启动日志可从后端当前内存快照同步，但不保存历史日志。启动会话支持停止单个启动会话或停止本次启动的全部会话。

项目归档只影响 Workbench 本地项目记录，不删除、移动或修改本地项目目录。运行中的项目不可归档，需要先停止本次启动会话。

项目详情包含：

- 项目名称
- 本地路径
- 备注
- 标签
- 启动配置

### 5.2 Skills 管理

Workbench 使用一个统一 Skills 根目录保存所有 skills。该目录是 skills 内容的唯一真实来源。

用户可以为每个 Skill 选择需要启用的全局工具，例如 Codex、Claude Code、OpenCode。启用后，Workbench 默认优先在对应工具的 skills 配置目录中创建指向统一根目录的符号链接；如果当前 Windows 会话无法创建符号链接，则自动复制完整 Skill 目录，并记录实际同步方式。

用户也可以为 Skill 选择需要启用的项目和工具。Workbench 在对应项目内受支持的工具 skills 目录中使用同一 Auto 方式启用，使 Skill 只在指定项目和工具中生效。不支持项目级 Skills 目录的工具不提供项目级启用选项。

功能需求：

- 配置一个 Workbench Skills 根目录。
- 扫描统一根目录下的 `SKILL.md` 文件。
- 解析 Skill 名称、描述和路径。
- 展示 skills 列表。
- 支持搜索 skills。
- 支持查看 skill 详情。
- 支持打开统一 Skills 根目录。
- 支持打开对应 `SKILL.md` 文件。
- 支持在 Skills 列表操作列打开对应 `SKILL.md` 文件。
- 支持为 Skill 选择全局启用工具。
- 支持在 Skills 列表中通过工具标识快速切换全局启用。
- 支持在 Skills 列表操作列删除 Skill，删除前必须确认。
- 支持为 Skill 选择项目及项目内启用工具。
- 支持对单个项目下当前 Skill 的所有工具进行全部启用或全部关闭。
- 支持创建和移除由 Workbench 管理的符号链接。
- Skill 默认使用 Auto 方式启用：优先创建符号链接，创建失败时自动复制到目标目录。
- Workbench 记录每个启用项实际使用的 Symlink 或 Copy 方式。
- 目标位置已有内容时不覆盖、不合并。
- 停用时只移除 Workbench 记录的符号链接或完整副本。
- 扫描时识别全局工具目录中的同名 Skill。
- 全局工具目录中同名 Skill 与 Workbench 根目录内容一致时，自动登记为 Workbench 管理。
- 全局工具目录中同名 Skill 与 Workbench 根目录内容不一致时，标记为冲突，由用户在 `.workbench`、`.codex`、`.claude`、`.opencode` 的可用版本中选择一个唯一版本源。
- 解决冲突前必须备份被替换版本到 `~/.workbench/backups/skills/<timestamp>/<tool>/<skill>`。
- 应用冲突解决后，选中的版本成为 Workbench 根目录中的唯一真实副本，已存在的全局工具目录统一重新同步。
- 支持删除 Skill。
- 删除 Skill 时删除统一根目录中的 Skill、分类记录和 Workbench 管理的启用记录。
- 删除 Skill 时只清理 Workbench 管理的符号链接或副本，不删除未被 Workbench 管理的工具目录内容。
- Copy 副本不会在扫描时自动覆盖；后续通过明确的重新同步操作更新。
- 支持自定义 Skill 分类。
- 每个 Skill 只能属于一个分类。
- 支持按分类筛选 Skills。
- 支持在 Skills 列表中双击分类标签编辑分类。
- 未分类 Skill 统一归入“未分类”。
- 分类只用于 Workbench 内整理，不影响文件目录和符号链接结构。
- 支持从 ZIP 文件导入 Skills。
- 支持从已解压文件夹导入 Skills。
- 导入时使用系统文件选择器或文件夹选择器。
- 一个 ZIP 文件或一个已解压文件夹中可以包含多个 Skills，并一次性导入。

导入流程：

- 用户选择 ZIP 文件或已解压文件夹。
- Workbench 扫描导入来源中所有包含 `SKILL.md` 的目录。
- Workbench 将所有可导入 Skills 复制到统一 Skills 根目录。
- 导入后的 Skills 默认不启用。
- 若统一根目录已存在同名 Skill，则停止导入该 Skill 并提示冲突，不覆盖、不合并。

MVP 不包含：

- 从在线来源安装 Skill
- RAR 导入
- Skill 在线市场
- 远程仓库管理

### 5.3 AI Radar

AI Radar 是本地 AI 信息条目库，用于记录 AI 项目、资讯、论文和其他信息。

MVP 阶段只做本地手动维护，不自动抓取、总结、评分或接入外部数据源。Workbench 在第一版中承担录入、展示、搜索、筛选、收藏和打开链接的职责。

分类固定为：

- 项目
- 资讯
- 论文
- 其他

功能需求：

- 添加 AI Radar 条目。
- 编辑和删除 AI Radar 条目。
- 展示条目列表。
- 支持搜索条目。
- 支持按分类筛选。
- 支持按标签筛选。
- 支持收藏条目。
- 支持查看条目详情。
- 支持打开条目链接。

条目字段：

- 名称
- 分类
- 链接
- 标签
- 备注
- 收藏状态
- 更新时间

MVP 只使用本地数据，不接入外部数据源。

### 5.4 设置

设置模块用于管理 Workbench App 的基础配置。

功能需求：

- 查看本地数据存储位置。
- 打开 Workbench 根目录、SQLite 数据库所在目录和 Skills 根目录。
- 管理 Workbench Skills 根目录。
- 查看支持的工具及其全局 Skills 目录。

## 6. 页面结构

Workbench App 采用左侧模块导航 + 右侧工作区。

左侧导航包含：

- 项目
- Skills
- AI Radar
- 设置

右侧工作区根据当前模块显示列表、详情、表单或状态面板。

第一版不做独立 Dashboard，避免和模块入口形成双入口。

## 7. 关键交互

### 7.1 添加项目

用户选择或输入本地项目路径，Workbench 生成项目记录。用户可以补充备注、标签和启动配置。

成功标准：

- 项目出现在项目列表中。
- 项目路径可以被打开。
- 项目详情可以查看和编辑基础信息。
- 项目记录可以归档和恢复，且不会影响本地项目目录。
- 配置启动项后，可以通过启动按钮在 Workbench 内为所有启用启动项创建本次启动会话。
- 项目本次启动输出可以进入启动日志详情页查看，并可在全部输出和单个启动项输出之间切换。

### 7.2 管理和启用 Skills

用户配置统一 Skills 根目录后，Workbench 扫描其中的 `SKILL.md` 并展示 Skills。用户可以为 Skill 设置分类，并选择需要全局启用或在指定项目中启用的工具。

成功标准：

- 能展示扫描到的 skill。
- 能搜索 skill。
- 能打开统一 Skills 根目录。
- 能打开 skill 文件。
- 能按分类管理和筛选 Skills。
- 能通过 Auto 同步在支持的全局工具目录中启用或停用 Skill。
- 能通过 Auto 同步在支持的项目工具目录中启用或停用 Skill。
- Workbench Skills 根目录始终保存唯一真实副本。
- 目标冲突时不会自动覆盖、合并或删除已有内容。
- 全局工具启用状态可以被识别为未启用、Workbench 管理或内容冲突。
- 内容冲突只能通过用户明确选择唯一版本源解决，且替换前会备份被替换版本。
- 能删除统一根目录中的 Skill，并清理 Workbench 管理的启用记录和受管目标。

### 7.3 导入 Skills

用户选择 ZIP 文件或已解压文件夹作为导入来源。Workbench 扫描其中所有包含 `SKILL.md` 的目录，并批量导入所有可导入 Skills。

成功标准：

- 能通过系统选择器选择 ZIP 文件或已解压文件夹。
- 能将来源中的多个 Skills 一次性复制到统一 Skills 根目录。
- 导入完成后 Skill 默认不启用。
- 同名 Skill 不覆盖、不合并，并明确提示冲突。
- 导入结果展示成功、跳过、无效数量，并支持打开统一根目录。

### 7.4 管理 AI Radar 条目

用户可以手动添加 AI Radar 条目，并通过搜索、分类、标签和收藏状态进行管理。

成功标准：

- 条目可新增、编辑、删除、查看、搜索和筛选。
- 收藏状态可切换。
- 链接可打开。
- 不依赖外部数据源也可以完整使用。

## 8. 非目标

MVP 明确不做以下内容：

- 不提供交互式终端、PTY、SSH、终端标签页或持久化启动日志。
- 不管理 AI Agent 配置、prompts、MCP 配置或规则文件。
- 不从在线来源安装 Skills。
- 不支持 RAR 导入。
- 不提供 Skills 在线市场或远程仓库管理。
- 不接入 AI Radar 外部数据源。
- 不提供 Obsidian 连接或笔记管理。
- 不做复杂插件系统。
- 不做多用户、登录、云同步。
- 不合并参考项目代码。
- 不把参考项目组成 monorepo。

## 9. 技术方向

第一版采用：

- Tauri
- React
- TypeScript
- SQLite

应用包含前端界面、本地 Tauri 后端能力和本地数据库。MVP 不启动独立 HTTP 后端服务。

本地数据建议放在系统标准 app data 目录中，源码目录只保存项目代码和文档。

## 10. 验收标准

MVP 完成时应满足：

- 可以启动 Workbench App 桌面应用。
- 左侧导航和基础页面结构可用。
- 可以添加、查看、搜索本地项目。
- 项目新增和编辑结果可持久化保存。
- 可以配置项目启动项，并通过启动按钮为所有启用启动项创建非交互式内嵌启动会话。
- 可以在项目模块内查看本次启动日志详情，不需要打开系统终端窗口。
- 可以归档和恢复项目记录，归档不会触碰本地项目目录。
- 可以配置一个统一 Skills 根目录并扫描 `SKILL.md`。
- 可以为 Skill 设置分类并按分类筛选。
- 可以通过 Auto 同步为全局工具启用或停用 Skill。
- 可以通过 Auto 同步为指定项目中的受支持工具启用或停用 Skill。
- 可以扫描全局工具目录中同名 Skill 的一致状态和冲突状态。
- 可以通过选择唯一版本源解决同一 Skill 的多个全局版本冲突。
- 可以删除 Skill，并且不会删除未被 Workbench 管理的工具目录内容。
- 可以从 ZIP 文件或已解压文件夹导入 Skills。
- 导入同名 Skill 时不会覆盖或合并现有 Skill。
- 可以添加、搜索、筛选 AI Radar 条目。
- 数据在应用重启后仍然保留。

## 11. 后续方向

MVP 稳定后，可以逐步考虑：

- 交互式终端、持久化启动日志和复杂项目进程编排。
- Agent 配置、prompts、MCP 配置和规则文件管理。
- Skills 在线安装、同步和远程仓库管理。
- AI Radar 导入式数据源，支持 JSON、Markdown、CSV 或现有 ai-radar 导出数据。
- 外部 collector 或 Codex automation 写入 radar inbox，再由 Workbench 统一导入。
- Obsidian 知识库连接。
- 自动化任务入口。
- 项目与 Agent 工作流绑定。
