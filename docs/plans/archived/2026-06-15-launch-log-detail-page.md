# 启动日志详情页计划

## 目标

将项目内嵌启动会话的日志展示从右侧详情面板迁移到项目模块内的日志详情页。项目列表页保留项目状态、行级操作和右侧摘要入口；点击查看详情后进入完整日志页面，使用顶部返回、会话 Tabs 和大面积日志区域展示本次启动输出。

## 范围

- 项目页右侧详情面板只展示启动摘要、状态和进入日志详情页的入口。
- 日志详情页位于现有 App Shell 的项目模块内，不新增独立主导航。
- 日志详情页展示本次启动的内存日志，支持“全部”和单个启动项 Tabs。
- 保留现有停止、重启、关闭本次记录等生命周期操作，并在日志详情页中给出合理位置。
- 更新与该体验直接相关的中文文档和设计规则。

## 非目标

- 不做交互式终端、PTY、输入命令或快捷键命令发送。
- 不保存历史日志；再次启动仍创建全新的会话组。
- 不照搬 Docker Desktop 的容器、镜像、端口、统计图等产品模型。
- 不引入新路由库或独立 HTTP 后端。

## 假设与已确认决策

- 用户已确认采用“顶部返回 + 会话 Tabs + 大日志区域”的 B 方案。
- 启动会话数据仍以当前前端 `launchRuns: Record<string, LaunchRun>` 为页面状态来源。
- 后端启动、停止、重启和输出事件机制先不改，除非实现中发现日志页需要暴露缺失字段。
- `DESIGN.md` 中旧的“已启动请求”和右侧详情日志承载规则需要同步更新为当前状态语义和日志详情页规则。

## 实施步骤

1. `done` 写失败测试：覆盖项目页点击查看日志后进入日志详情页、顶部返回回到项目列表、日志页按“全部/单会话”展示输出。
   - 验证：`pnpm test -- --run src/App.test.tsx` 已先出现预期失败，随后实现后通过。

2. `done` 调整项目模块页面状态：在 `ProjectsView` 内增加轻量页面模式，用选中的 `LaunchRun` 进入日志详情页；不引入全局路由。
   - 验证：项目列表和现有筛选、选择、启动/停止行为不回退。

3. `done` 改造右侧启动摘要：移除完整日志输出区域，保留会话状态摘要、停止/重启/关闭本次记录，以及“查看日志”入口。
   - 验证：右侧面板在窄空间下不再显示大块日志，按钮图标和状态语义保持可读。

4. `done` 新增日志详情页组件：顶部返回和项目/启动组信息，Tabs 展示“全部”和各启动项，主体为大面积日志输出区。
   - 验证：多启动项、部分停止、失败、已停止、运行中等状态均可在 Tabs 和日志区正确表达。

5. `done` 同步样式和设计文档：保持现有深浅色 token、紧凑工作台风格；更新 `DESIGN.md` 中项目启动状态和日志详情页规则。
   - 验证：无文字溢出，路径和命令安全换行；文档不再保留旧“已启动请求”作为推荐规则。

6. `done` 完整验证：运行前端测试、必要时用浏览器检查深浅色下项目页和日志详情页；最后运行 `pnpm verify`，如 Tauri release 构建被运行中的 exe 占用则单独说明。
   - 验证：前端定向测试已通过；完整验证结果记录在分支收尾中。

## 风险

- `src/App.tsx` 已承担较多页面逻辑，新增日志详情页时应优先局部组件化，避免扩大重构范围。
- 右侧摘要和日志详情页会共享启动控制操作，需避免出现两个入口状态不一致。
- “全部日志”需要合并多个 session 输出；第一版可按当前 session 输出数组顺序拼接，不尝试精确跨进程时间排序，除非后端事件已有可靠时间戳。

## 验收标准

- 从项目列表或右侧摘要可以进入某项目本次启动的日志详情页。
- 日志详情页有返回入口，返回后项目列表状态不丢失。
- 日志详情页可查看全部日志，也可切换到单个启动项日志。
- 右侧详情面板不再用小空间展示完整日志。
- 启动、停止、部分停止、单项重启、全部重启和关闭本次记录的状态展示保持一致。
- 不新增交互式终端能力，不保存历史日志。
