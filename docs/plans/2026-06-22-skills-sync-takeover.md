# Skills 同步与工具目录接管计划

## 目标

将本地 Skills 页的“扫描”和“发现已有工具 Skills”收敛为一个“同步 Skills”入口：先刷新 Workbench 统一根目录，再只读发现外部工具目录 Skills；当用户确认同步外部项时，导入到统一根目录并把对应工具目录目标接管为 Workbench 管理的链接或副本，接管前必须备份原工具目录内容。

## 范围

- 将本地 Skills 顶部操作从“扫描”和“发现已有工具 Skills”调整为一个“同步 Skills”入口。
- 同步入口执行两个阶段：
  - 刷新当前统一根目录，更新 Workbench 已管理 Skills 列表。
  - 发现已注册工具全局目录中的外部 Skills，并按状态进入用户决策。
- 对用户确认的外部项执行导入与接管：
  - 外部有、Workbench 没有：复制到统一根目录，备份工具目录原内容，再把工具目录目标替换为 Workbench 管理链接或副本，并写入启用记录。
  - 外部有、Workbench 有且内容相同：作为无待处理项处理；若没有新增、冲突、不可用项，不打开弹窗，只通过 toast 汇总。
  - 外部有、Workbench 有且内容不同：作为冲突，必须由用户选择版本来源后才能导入或接管。
- 同步发现阶段必须保持窗口可响应：点击后只让“同步 Skills”按钮进入 loading 状态，发现完成后再按结果决定 toast 或弹窗。
- 同步弹窗只承载需要用户处理的项：新增、冲突、不可用或不可读；footer 只保留一个“同步”主按钮，关闭由右上角 X 完成。
- 同步结果要清楚展示导入、接管、跳过、冲突和失败状态。
- 删除、停用、重建等后续行为继续只处理 Workbench 已记录的受管目标。
- 更新文档、类型、API、测试和变更日志。

## 非目标

- 不在应用首次启动时自动同步或自动接管工具目录。
- 不扫描未注册工具目录。
- 不接管用户跳过的外部 Skills。
- 不自动处理同名不同内容冲突。
- 不把项目级工具目录纳入本次同步；本次只处理已注册工具的全局 Skills 目录。
- 不改变 skills.sh 市场安装和更新语义；市场安装后仍默认不启用到工具目录。
- 不引入“外部已启用但未受管”的长期列表状态；跳过项不进入 Workbench 管理模型。
- 不删除未被 Workbench 接管或记录的工具目录内容。

## 假设与决策

- 决策：用户点击“同步 Skills”后，默认语义是“刷新本地真实来源 + 发现外部工具 Skills”，但只有在弹窗中确认外部项后才执行导入和接管。
- 决策：同步外部工具目录到 Workbench 根目录时，用户确认的项默认表示接受 Workbench 接管对应工具目录目标。
- 决策：接管必须先备份工具目录原目标，再删除或替换原目标，并通过现有 Auto 同步创建指向统一根目录的链接或 Copy 副本。
- 决策：同步发现期间不先打开弹窗；顶部“同步 Skills”按钮显示 loading、禁用重复点击，刷新图标旋转。
- 决策：发现结果只有已存在相同内容项，且新增、冲突、不可用、不可读均为 0 时，不打开弹窗，toast 显示“Skills 已同步，无待处理项；N 项已存在相同内容”。
- 决策：发现结果没有任何外部候选时，不打开弹窗，toast 显示“Skills 已同步，未发现外部工具 Skills”。
- 决策：同步新增项可以默认勾选；同名相同内容项不进入主要操作、不默认接管、不修改工具目录，只在存在其他待处理项时作为弱化折叠信息展示。
- 决策：冲突项默认不勾选且必须选择版本来源。
- 决策：冲突项支持三类动作：保留 Workbench 版本并接管工具目录、使用外部工具目录版本覆盖 Workbench 后接管、跳过。
- 决策：多个工具目录中出现同名外部 Skill 时按工具目标逐项接管；同名不同内容必须以 Skill 为单位先确认唯一 Workbench 版本，再接管被选中的工具目标。
- 决策：接管后的启用记录写入 `skill_enablements`，后续停用和删除按现有 Workbench 受管目标规则清理。
- 决策：同步弹窗必须在确认前明确说明“将修改工具目录，并在替换前备份原内容”。
- 决策：同步弹窗 footer 不提供“取消”“跳过并关闭”“全部跳过”；右上角 X 表示关闭不处理，主按钮固定文案为“同步”。
- 决策：“同步”按钮在没有可执行选择或冲突未解决时禁用；有可执行选择且冲突选择完整时启用。
- 决策：“重新扫描”不放在 footer，放到“当前统一根目录”信息行右侧或同一区域；loading 只通过按钮图标旋转和文案变化表达，不在根目录下额外增加扫描说明。
- 决策：同步弹窗主体使用 `新增 / 冲突 / 已存在 / 不可用` 状态 Tab；顶部不重复放统计卡片。
- 决策：新增 Tab 顶部只保留一个“导入并接管”toggle；默认选中，再次点击取消选择并表示不处理。
- 决策：冲突 Tab 默认不选择版本来源，只提供“全部保留 Workbench”和“全部使用外部”两个可取消批量 toggle；批量策略按钮使用操作按钮组视觉，不伪装成 Tab 或分段栏。
- 决策：安全提示放在 footer 摘要下方，避免占据首屏核心展示区域，但仍在用户点击同步前可见。
- 决策：同步弹窗使用固定高度，头部、当前根目录、状态 Tab 和 footer 不参与滚动；候选列表较多时只滚动当前 Tab 列表。
- 决策：空 Tab 使用轻量空状态，不使用大面积填充色块铺满内容区。
- 决策：提交同步时主按钮显示“同步中”并禁用重复提交、重新扫描和候选选择；成功后关闭弹窗、刷新 Skills 列表和启用状态，只用 toast 汇总结果；失败、冲突或无效项才保留可读原因。
- 决策：后端耗时的目录扫描、内容比较、复制、备份和接管必须放入阻塞线程或等价后台执行路径，避免 Tauri 窗口进入“未响应”。
- 决策：提交同步时若重新发现同名相同内容，后端返回跳过且不接管；若重新发现同名不同内容，返回冲突并要求重新扫描选择版本来源。
- 决策：不需要新增 ADR；这是既有 Skills 数据所有权规则的产品语义收敛，不改变统一根目录作为唯一真实来源的长期架构。

## 已知不确定性

- Windows 上创建符号链接可能失败，接管仍需沿用现有 Auto 同步回退 Copy 行为。
- 多来源同名 Skill 的 UI 选择需要保持紧凑，避免弹窗过于复杂；实现时应优先使用可折叠或分组列表，而不是在表格里塞满操作。
- 当前导入与冲突解决已有备份逻辑，但“接管工具目录目标”的备份结果需要在返回值中显式暴露，便于 UI 展示和测试断言。
- 当前同步发现会造成窗口短暂“未响应”，需要通过后端后台化和前端按钮 loading 共同验证。
- 当前工作区已有未提交的沙盒计划文档；实施本计划时应单独开分支并避免混入无关变更。

## 模块与代码放置约束

- 后端 command facade 保持在 `src-tauri/src/skills.rs`，复杂逻辑优先放入 `src-tauri/src/skills/` 下具名 owner 模块。
- 外部同步和接管策略应放入导入或同步职责清晰的模块，例如扩展 `importer.rs` 或新增 `sync.rs`；不得新增 `utils`、`helpers` 等空泛模块。
- 文件替换、备份、链接和 Copy 操作复用 `filesystem.rs` 中已有函数；如需新增函数，函数名必须表达明确副作用。
- 工具目标路径和启用记录继续复用 `tool_targets.rs`、数据库保存函数和现有 `SyncMethod`。
- 前端弹窗应放在 `src/components/dialogs/skills/`，Skills 页编排保持在 `src/views/skills/SkillsView.tsx` 或现有 app-level 状态边界中。
- 不因本任务继续拆分 `App.tsx`；若实施时发现大量 app-level 状态新增，应优先把弹窗内部选择状态放入 Skills dialog 组件。

## 边界情况清单

- 外部工具目录不存在：同步发现跳过，不创建目录。
- 外部工具目录为空：同步完成后不弹窗，toast 显示未发现外部工具 Skills。
- Workbench 根目录不存在：按现有扫描逻辑创建或初始化统一根目录。
- 外部 Skill 缺少 `SKILL.md`：标记无效，不导入、不接管。
- 外部 Skill 目录名非法：标记无效，不导入、不接管。
- 外部有、Workbench 没有：默认导入并接管，备份原工具目录目标。
- 外部有、Workbench 有且内容相同：不接管、不修改工具目录；若没有其他待处理项，只显示无待处理 toast。
- 外部有、Workbench 有且内容不同：必须选择版本来源；未选择不能确认。
- 多个工具目录同名同内容：不写入启用记录；若没有其他待处理项，只在 toast 中汇总数量。
- 多个工具目录同名但内容不同：必须先确定 Workbench 唯一版本，再接管被选工具目标。
- 工具目录目标是已有符号链接且不指向 Workbench：备份链接信息或链接目标可恢复后再替换；不能直接当作 Workbench 管理目标。
- 工具目录目标是普通文件：不删除，返回失败或冲突。
- 备份失败：不删除、不替换、不写启用记录。
- 创建符号链接失败：回退 Copy，并记录 `sync_method = copy`。
- Copy 回退失败：保留备份和原始失败信息，不写启用记录。
- 用户跳过：不导入、不接管、不写启用记录。
- 用户点击 X 关闭弹窗：只完成根目录刷新和只读发现，不修改工具目录。
- 弹窗无可执行选择：主按钮“同步”禁用，不提供“全部跳过”作为提交动作。
- 同步发现耗时较长：按钮保持 loading，窗口仍应可移动和响应点击，不应出现系统“未响应”。
- 重新扫描耗时较长：只通过“重新扫描”按钮自身的旋转图标和“扫描中”文案反馈，避免增加额外说明行。
- 当前 Tab 无内容：显示轻量空状态，不用大块色块铺满列表区域。
- 同步提交耗时较长：主按钮显示“同步中”并旋转，重新扫描、候选勾选和策略切换禁用，防止重复提交。
- 外部候选较多：弹窗外层不滚动，只滚动当前状态 Tab 的列表区域。
- 同步成功：关闭弹窗，刷新主 Skills 列表和工具启用状态，通过 toast 汇总接管和跳过数量。
- 同步出现失败、冲突或无效结果：保留弹窗并展示可读原因，便于用户修正或重新扫描。
- 删除同步接管后的 Skill：只清理接管后写入记录的目标，符合用户“已由 Workbench 管理”的预期。
- 停用同步接管后的 Skill：只清理对应受管目标，不删除 Workbench 根目录真实副本。
- skills.sh 来源 Skill：若外部同步覆盖统一根目录，需要清理或保留来源记录的策略必须明确；计划建议外部覆盖后清理 `skill_sources`，避免市场更新状态指向错误来源。
- 自定义工具目标：纳入同步发现和接管；删除自定义工具仍不删除外部目录，只清理配置和启用记录。

## 验收标准

- 本地 Skills 页顶部不再同时展示“扫描”和“发现已有工具 Skills”，改为清晰的“同步 Skills”入口。
- 点击“同步 Skills”会刷新 Workbench 统一根目录，并检查外部工具目录；执行期间按钮进入 loading，窗口不进入“未响应”。
- 没有新增、冲突、不可用或不可读项时不打开弹窗，只显示对应 toast。
- 有待处理外部候选时，弹窗明确提示同步会导入并接管工具目录目标，接管前会备份。
- 弹窗 footer 只保留一个“同步”主按钮；没有可执行选择或冲突未解决时禁用。
- 弹窗使用状态 Tab 展示新增、冲突、已存在和不可用项；已存在和不可用为只读。
- 弹窗默认高度能容纳主要操作区域；候选较多时只滚动当前列表，不让外层页面或弹窗主体滚动。
- 空 Tab 使用轻量空状态，不出现大面积提示色块。
- 批量策略按钮呈现为操作按钮组，不与状态 Tab 混淆。
- 点击弹窗内“同步”后主按钮显示 loading，重复同步、重新扫描和候选选择暂时禁用。
- 冲突批量策略默认不选，点击“全部保留 Workbench”或“全部使用外部”可批量选择，再次点击可取消。
- 安全提示显示在 footer 摘要下方，不占据顶部核心展示区域。
- 新增项可通过单个“导入并接管”toggle 批量选择或取消，接管后对应工具显示为 Workbench 管理启用；同内容项不进入同步操作。
- 同名不同内容冲突不会自动处理，必须用户选择版本来源。
- 提交时若候选状态变化为同名相同内容，后端自动跳过，不接管工具目录且不显示失败。
- 接管前备份原工具目录目标，备份失败时不替换原目标。
- 同步成功后自动关闭弹窗并刷新 Skills 列表；不展示成功后的逐项结果列表。
- 同步后删除或停用 Skill 只影响 Workbench 接管并记录的工具目录目标。
- 跳过项不会进入 Workbench 管理模型。
- 自动化测试覆盖后端核心同步接管、备份失败保护、冲突选择、前端同步弹窗和入口合并。
- `pnpm verify` 通过。
