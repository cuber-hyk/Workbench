# Workbench 设计系统

## Authority And Scope

本文档记录 Workbench 已确认的 UI 方向，是设计意图、布局规则、可复用 UI 模式和禁用模式的来源。

精确视觉值保存在 `design-tokens.json`。当前代表性视觉来源是 `UI/` 中的静态原型。

## 2. 已确认的产品 UI 方向

Workbench 使用本地优先的桌面工作台界面。整体体验应接近实用型开发控制台：安静、紧凑、清晰，适合日常高频使用。

Workbench 不是营销站、落地页、知识图谱或装饰性 Dashboard。首屏应直接呈现真实可用的工作台模块。

## 3. App Shell

Workbench 使用固定左侧导航 + 右侧工作区。

已确认导航模块：

- 项目
- Skills
- 资源 Radar
- 设置

规则：

- 左侧边栏是持久模块切换入口。
- 右侧工作区根据当前模块切换内容。
- MVP 不设置独立 Dashboard 模块。
- 当有助于表达本地工作台定位时，侧边栏可以展示本机工作区状态。
- 主题切换属于 App Shell 和设置模块。
- 保留系统原生标题栏；应用内容顶端只使用细分隔线、浅高光和微阴影处理标题栏与内容区的衔接，不自绘窗口按钮。
- 侧边栏品牌标识使用 Workbench 图标资产和 `Workbench` 文本，不使用文字 `W` 作为长期品牌标记。

来源：

- `docs/PRD.md`
- `docs/ARCHITECTURE.md`
- `UI/index.html`
- `src/assets/brand/workbench-icon.svg`

## 4. 布局模式

### 4.1 模块页面

每个主模块页面使用：

- 页面标题区，包含标题、简短上下文说明和必要的主操作。
- 工具栏，用于搜索和筛选。
- 主内容区。

高度规则：

- App Shell 使用固定 `100dvh` 高度。
- 一级模块页面标题区统一使用 `design-tokens.json` 中的页面标题高度；设置、项目、Skills 等页面不能因操作数量或自定义布局产生不同高度。
- Skills 在窄窗口下允许子视图 Tab 换行，此时标题区可以按内容增高，不裁切标题、Tab 或操作。
- 页面外层不产生滚动条。
- 页面内滚动容器保留滚动能力，但默认隐藏可见滚动条。
- 列表和表格布局应优先通过可压缩列宽、文本截断和操作列预留空间避免横向滚动。
- 列表区、详情区和设置页主体是明确的滚动容器。
- 主内容区使用 flex 和 `min-height: 0` 控制高度，不依赖硬编码 `calc(100vh - ...)`。

### 4.2 列表 + 详情

项目、Skills 和资源 Radar 视图使用列表 + 详情的工作模式。

规则：

- 列表是模块内部的主要导航区域。
- 详情面板展示当前选中项及其相关操作。
- 编辑可以在详情面板中以表单字段呈现，也可以使用聚焦型弹窗。
- 搜索和筛选位于列表上方，不放在详情面板内部。

### 4.3 设置

设置页面使用专业软件偏好设置布局：页面内部左侧是设置分类导航，右侧是当前分类的单一表单式内容区。分类导航用于常规、Skills、工具目录、项目打开方式、本地数据、应用行为和外观等主题，不新增 App Shell 主导航项。

设置页右侧内容不使用多个大卡片堆叠。当前分类内容通过页面标题、分区标题、简短说明、细分隔线和紧凑设置行组织；只有迁移检查结果、冲突提示、危险确认等需要聚焦或状态承载的内容才使用面板化视觉。

设置页内容区在默认窗口和全屏窗口下使用响应式宽度，宽屏时扩展到合理最大宽度但不无限拉长。分类导航和右侧内容区都是内部滚动容器，保留滚动能力但隐藏可见滚动条，避免深色主题下滚动条抢占注意力。

设置页路径行使用左侧名称和路径、右侧操作与状态聚合的布局。路径输入旁的打开目录图标必须和输入框同高同线。

支持的工具目录设置行可以提供上移 / 下移图标按钮调整 Skills 表格的全局工具展示顺序；排序控制与打开目录、可用状态保持在同一行的右侧操作区。打开目录时若目标目录不存在，先弹出确认，由用户决定是否创建并打开。

项目打开方式使用设置页分类内容管理。每个打开方式以紧凑表格式行展示名称、类型、命令摘要、启用状态和编辑/删除操作；编辑放入聚焦弹窗，避免设置页行内表单撑坏布局。

应用行为设置使用单独分类管理。关闭窗口行为使用下拉选项表达为“隐藏到托盘 / 退出应用”，不与主题或本地数据混在同一设置行。

## Component Rules

### 5.1 按钮

按钮类型：

- 默认按钮：用于次要操作。
- 主按钮：用于页面或弹窗的主要操作。
- 危险按钮：用于删除等破坏性确认操作。
- 图标按钮：用于编辑、关闭等紧凑操作。
- 全宽按钮：用于详情面板中明确的局部主操作。

规则：

- 按钮高度保持紧凑。
- 操作使用图标或短标签。
- 不使用营销式的大型 CTA 按钮。
- 行级图标操作使用统一操作组，点击时不能触发行选择。
- 删除按钮必须使用 danger 语义，并进入确认弹窗。
- 只有会触发动作的控件使用 Button 或 IconButton；稳定状态、不可安装、不可更新、已完成等只读信息使用状态徽标或 muted 文本，不使用按钮视觉。
- 同一区域内的文字按钮应保持一致的高度、字号、图标尺寸和内边距；图标按钮只用于紧凑单动作，不和文字按钮混用来表达同一种操作。
- 页面顶部和工具栏的文字按钮使用 15-16px lucide 图标；详情面板中的紧凑跳转、编辑、打开类动作使用统一 IconButton 尺寸。

### 5.1.1 路径输入

路径输入保留可手动编辑的文本框，并在同一行右侧提供文件夹图标按钮打开系统目录选择器。

规则：

- 项目路径和启动工作目录使用相同的输入 + 图标按钮布局。
- 文件夹图标按钮使用固定宽度，不挤压路径输入框。
- 选择目录后直接回填对应字段；取消选择不改变现有内容。

### 5.2 Row Card

Row Card 用于项目和 Radar 列表项。

规则：

- Row Card 展示标题、状态或分类元信息，以及可选标签或简短备注。
- 选中态和悬停态使用同一语义强调色系。
- Row Card 是可点击列表项，不是装饰性内容卡片。

### 5.3 Table Row

当模块需要横向比较多项属性时使用 Table Row，例如项目的标签、启动项和启动状态，以及 Skills 的分类和全局启用状态。

规则：

- 在滚动列表中，表头可保持 sticky。
- 表头与列表使用相同面板背景，通过文字层级和底部分隔线区分，不使用独立灰色色块。
- 单元格内容应安全截断或换行，不能撑坏布局。
- 工具启用标识保持紧凑。
- 项目列表使用表格展示项目、标签、启动项、启动状态和操作。
- 项目列表中的打开目录、启动和编辑属于行级高频操作，应放在操作列。
- 项目列表中的启动项、启动状态和操作列表头必须与对应列内容左对齐。
- 项目列表中的“用工具打开”属于行级高频操作，使用单个图标菜单承载 VS Code、Trae、PowerShell、Claude Code 等 Profiles，不把所有工具图标平铺在表格行内。
- 项目列表行级操作在默认窗口宽度下必须完整可见；空间不足时优先压缩或隐藏低优先级信息列，不隐藏操作列。
- 项目列表启动状态只表达项目级摘要：未配置、可启动、运行中、部分运行、失败、已停止、已结束。
- 项目归档状态使用紧凑标签表达；归档和恢复属于行级操作，可放在项目表格操作列。
- 项目启动主操作执行所有启用启动项；本次启动会话支持停止或重新启动单个启动项。
- 项目打开方式不属于启动项；GUI 应用直接打开项目目录，交互式 CLI 通过外部终端打开，不进入 Workbench 内嵌启动日志。
- Skills 列表中的全局工具标识可以直接点击切换启用状态；内容冲突状态只展示状态，不直接切换。
- 全局工具标识使用紧凑彩色工具图标；缺少正式图标资源时才使用短字母 fallback。
- 当全局工具数量超过表格可读宽度时，Skills 列表只展示用户排序中的前 4 个工具图标和 `+N` 入口；`+N` 使用小浮层展示全部工具并保留切换能力。
- Skills 表格在默认窗口宽度下必须完整展示操作列按钮；分类和全局启用列可适度收窄，Skill 描述使用截断，不应产生横向滚动条。
- Skills 顶部工具栏支持按分类、状态、启用工具和启用项目筛选；工具筛选同时匹配全局启用和项目级启用。
- Skills 使用 `本地 Skills / 技能市场 / 更新` 内部子视图；子视图切换使用紧凑文本 tab，不新增左侧主导航项。
- Skills 页标题、子视图 tab 和页面操作在常用窗口宽度下保持同一顶部行；只有当宽度不足以保证左右区域可读时，tab 才迁移到下一行居中展示。
- Skills 子视图的动作按归属放在各自内容区，不把子视图专属动作常驻在 Skills 顶部，也不混入筛选工具栏。`同步 Skills`、`管理分类` 和 `导入 Skills` 只属于本地 Skills，并放在统一根目录管理区；技能市场使用自己的搜索、状态筛选和刷新市场动作；更新页使用自己的检查更新和更新动作。Skills 顶部只负责页面标题、上下文和子视图切换。
- 本地 Skills 页的“同步 Skills”是分阶段主操作：点击后按钮进入 loading，先刷新统一根目录，再检查外部工具目录；只有存在新增、冲突、不可用或不可读项时才打开聚焦弹窗。仅发现同名相同内容项或没有外部候选时使用 toast 汇总，不打开弹窗。
- 技能市场和更新页继续使用列表 + 详情工作模式；市场列表使用表格行展示远程 Skill、来源、安装状态、下载量和操作，更新列表使用表格行展示选择框、本地版本、远端状态、最近检查和操作。
- 本地 Skills、技能市场和更新页复用相同的操作列表头左对齐规则，但控件尺寸按页面自适应；本地 Skills 保持紧凑图标按钮，技能市场和更新页使用文字按钮。
- 技能市场同一操作列内的安装、卸载、安装中和不可安装状态必须保持相近高度、字号、图标尺寸和间距；只有真实动作使用 Button，不可安装等稳定状态使用状态徽标视觉。
- 技能市场列表的状态列使用 SkillStatusIndicator 表达安装状态；操作列只放安装、卸载、更新等命令，或不可安装这类 muted 只读动作占位。
- 技能市场详情区的来源跳转使用 IconButton；详情区不重复放安装、卸载或不可安装按钮，避免和列表操作列形成双入口。
- 技能市场在筛选栏下使用紧凑统计条展示全部、已安装、未安装、可更新、不支持和当前结果；详情区只提供来源跳转，安装和卸载入口保留在列表操作列。
- 更新页选择框只对可更新项启用；非可更新项和更新执行中的条目使用禁用态，鼠标显示不可操作语义是预期反馈。
- 桌面应用默认窗口宽度应优先满足 Skills 表格操作列完整展示；响应式列宽仍必须保证窄窗口下操作列不被裁切。
- 技能市场初次加载使用列表与详情骨架屏，不使用大块色面或解释性空卡片。
- 技能市场安装状态使用语义图标标签，至少区分未安装、已安装、可更新、不支持和检查失败；“已安装”和“不支持”作为状态展示，不作为按钮。
- 市场安装使用列表按钮内的百分比和细进度条反馈，百分比来自后端安装阶段事件，代表阶段完成度而非下载字节的精确进度，不额外显示阶段说明文字。
- Skills 列表中的分类使用下拉控件选择已有分类或创建新分类；分类选项来自统一分类表，“未分类”是不可删除的系统分类。
- Skills 分类管理使用聚焦弹窗，支持新增、重命名、删除迁移和合并分类；详情面板不重复展示分类信息或分类管理入口。
- Skills 分类管理弹窗中的分类列表使用紧凑表格行，不使用大卡片；列结构为分类名、Skills 数量和操作，操作按钮横向排列。
- Skills 分类管理的删除或合并确认区在分类列表上方展开，目标分类选择和确认按钮保持同一行，避免用户滚到底部确认。
- `未分类` 在分类管理中显示为系统分类标签，删除、重命名和合并操作保持禁用。
- Skills 列表中的操作列承载打开 `SKILL.md` 和删除图标按钮；删除点击后必须弹出确认弹窗。
- 资源 Radar 使用 Row Card 列表；收藏和删除属于行级操作，放在 Row Card 标题右侧，删除前必须弹出确认弹窗。
- 资源 Radar Row Card 元信息展示资源类型、领域、来源、语言、Stars 数量和更新时间；收藏使用右侧星标图标，不在元信息里写“已收藏”。
- 资源 Radar 的来源、语言、Stars 数量、来源失效状态和待合并重复状态使用文字明确展示，不能仅依赖颜色。
- 资源 Radar 顶部工具栏默认只展示搜索、类型、领域、来源、更多筛选和收藏星标，避免默认窗口下筛选项铺满两行。
- 资源 Radar 的语言、来源状态和重复状态放入“更多筛选”；标签和 GitHub Topics 只参与搜索，不提供独立下拉筛选。
- 资源 Radar 收藏星标使用 attention 色表达已收藏，未收藏使用 muted 色。
- Skills 操作列宽度使用 `skillsActionColumnWidth` token，避免按钮换行。
- 项目、Skills 和资源 Radar 的行级操作使用统一 `ActionGroup` / `row-actions` 模式；按钮点击必须阻止冒泡到行选择。
- 项目和本地 Skills 只有在记录包含可信第三方 `sourceUrl` 时才在详情面板显示“查看来源”图标；详情使用只读 URL + IconButton 的 `field-with-action` 模式。列表操作列不重复放来源跳转，没有可信来源时不显示按钮或空占位。
- 项目、Skills、技能市场、更新页和资源 Radar 的长列表使用统一紧凑分页控件；分页控件属于列表面板，固定在面板底部显示当前范围、总数、每页数量、上一页 / 下一页和可编辑当前页，不占用页面标题、筛选区或统计卡片。用户可输入合法页码范围内任意页并跳转，超出范围自动收敛到最近合法页；筛选或搜索变化后回到第一页，翻页后详情面板切换到当前页可见项。

### 5.4 详情面板

详情面板用于呈现选中项。

规则：

- 顶部显示主名称和次级元信息。
- 标题区使用统一详情标题模式；标题右侧可放编辑、归档、收藏等紧凑标题操作。
- 详情使用表单式字段或小型标签块。
- 文件路径和命令使用等宽字体，并且必须安全换行。
- 重要边界说明使用紧凑提示块。
- Skill 出现多个全局版本冲突时，使用 Skill 级冲突面板统一选择唯一版本源。
- 冲突面板必须先说明状态，再展示 `.workbench` 和存在版本的全局工具目录选项，最后展示应用和打开目录等辅助操作。
- 资源 Radar 出现 GitHub Stars 重复组时，在列表上方使用紧凑冲突面板展示候选资源；用户直接选择主资源并执行合并，不使用额外弹窗。
- 资源 Radar 收藏入口保留在列表 Row Card 星标；详情页不重复展示收藏星标。
- 资源 Radar 详情标题右侧展示编辑图标；打开链接图标放在链接字段右侧；详情页不重复展示删除入口。
- 资源 Radar 详情底部不再放常用文字按钮，避免默认窗口宽度下按钮挤压。
- 详情底部操作区使用统一 primary / secondary / danger 分组；危险操作与常用操作保持视觉分离。
- 全局工具启用只在列表表格中操作，详情面板不重复展示全局工具启用区。
- 本地 Skills 详情面板不展示项目启用摘要；Skills 与项目关系管理后续进入独立设计。
- 打开文件和删除 Skill 属于行级操作，放在表格操作列；详情面板只展示路径信息，不重复放操作按钮。
- 项目详情面板展示项目字段、备注、最近启动时间和统一启动项列表；启动项列表合并配置与本次会话状态，卡片右上角展示状态，底部只保留必要的停止或重启动作。
- 本次启动会话状态来自当前内存中的启动会话组，不代表历史运行记录。
- 项目详情面板不展示完整启动日志；启动摘要提供查看日志入口，进入项目模块内的日志详情页。
- 启动日志详情页使用统一高度的顶部标题区，标题同行展示启动组状态和时间摘要，右侧保留返回与会话操作；不脱离现有 App Shell，不新增主导航。
- 启动日志区域占满详情页剩余空间，使用紧凑 Tabs 顶栏和深灰黑底高密度日志排版；终端区域不使用卡片圆角，日志中的 HTTP / HTTPS 地址可点击并交由系统浏览器打开。

### 5.5 弹窗

弹窗用于创建、导入等聚焦任务。

规则：

- 弹窗只承载当前任务。
- 导入来源选择优先使用系统文件选择器或文件夹选择器。
- 导入弹窗展示导入摘要、状态列表、导入边界提示和最终操作。
- Skills 外部同步弹窗使用状态 Tab 展示 `新增 / 冲突 / 已存在 / 不可用`；新增项默认导入并接管，新增 Tab 顶部只保留一个“导入并接管”toggle，再次点击表示取消选择并不处理；冲突项默认不选择版本来源且必须选择后才能同步，已存在和不可用 Tab 只读。批量策略按钮属于操作按钮组，不使用 Tab 或分段控件视觉。弹窗顶部不放大段提示，安全提示放在 footer 摘要下方；footer 只保留一个“同步”主按钮，重新扫描放在当前统一根目录信息行，loading 只通过按钮图标旋转和文案变化表达。同步弹窗应使用固定高度，只有当前 Tab 的候选列表滚动；空 Tab 使用轻量空状态，不铺满内容区；提交同步时主按钮显示 loading 并冻结重复操作，成功后关闭弹窗、刷新列表并用 toast 汇总。
- 导入完成后可以提供打开统一根目录入口。
- 弹窗头部和底部操作区保持固定，只有内容区滚动。
- 软件更新弹窗中的 Release Notes 必须解析为版本标题、分类统计和分组列表；当当前版本落后多个版本时，默认展示当前版本到最新版本之间的累计更新说明，多版本说明使用版本折叠分组，最新版本默认展开，历史版本默认收起，版本标题在更新说明滚动区内吸顶；不直接展示 `##`、`###`、列表符号等 Markdown 原文。
- 破坏性或冲突相关信息必须在确认前可见。
- 删除确认使用统一确认弹窗模式，footer 顺序为取消在左、危险确认在右。
- 关闭窗口首次提示弹窗只说明“窗口将隐藏到托盘”，底部保留取消和确认操作；退出应用不在该弹窗中作为并列选项，用户可通过托盘菜单或设置页配置退出行为。

### 5.6 状态徽标

状态徽标用于表达项目状态、启动会话状态、Skill 冲突、启用状态、来源失效和待合并状态。

规则：

- 状态徽标使用统一 `StatusBadge` 语义，不按模块各自定义颜色规则。
- 状态语义包括 neutral、accent、success、warning、danger 和 attention。
- 状态信息必须保留文字，不允许只用颜色表达。
- 运行中、启用等正常积极状态使用 success 或 accent；失效、失败、冲突使用 danger；待处理但非错误状态使用 warning。

### 5.6.1 Toast 通知

Toast 用于非阻断反馈，固定显示在右下角，不占用模块主内容空间。

规则：

- Toast 支持 neutral、success、warning 和 danger 语义，不能把所有状态都做成同一种深色提示。
- Toast 必须同时适配浅色和深色主题，使用现有主题变量而不是硬编码单主题色。
- 需要用户处理但不阻断应用的状态使用 warning，例如缺少 GitHub CLI 或尚未登录。
- Toast 内容应包含语义图标、可读文案和关闭按钮；可操作命令使用等宽 code chip 展示。
- Toast 不替代确认弹窗；删除、覆盖、退出等高副作用操作仍使用确认弹窗。

### 5.7 主题

Workbench 支持浅色和深色主题。

规则：

- App Shell 和设置中的主题状态应保持一致。
- 两种主题使用相同布局和组件层级。
- 深色主题不是另一套视觉方案，而是同一工作台规则在深色 token 下的表达。
- 浅色主题必须是完整浅色主题，App Shell、左侧主导航、本机工作区状态、主题切换按钮和更新提示都使用浅色 sidebar token，不保留大面积深色侧边栏。
- 侧边栏颜色使用 sidebar 语义 token 表达，包括背景、active、hover、surface、control、text、muted 和 border；禁止在 App Shell 关键样式中硬编码单主题深色。

## 6. 文案规则

规则：

- UI 默认使用中文。
- 技术标识、路径、命令、API 名称和工具名称可以保留英文。
- 页面文案应简洁、操作导向。
- 避免用可见文案解释显而易见的 UI 机制。
- 当文案描述重要行为边界时，可以使用边界提示，例如启动会话日志只保存在当前内存中。

## Foundations

已确认基础：

- 紧凑的桌面优先布局。
- 克制的中性色界面，以 teal 作为主强调色。
- orange 仅用于收藏或注意状态。
- 使用边框区分面板和列表行。
- 圆角保持较小，通常为 4px 到 8px。
- 字体使用系统 UI 字体，并包含 Microsoft YaHei fallback。
- 路径、命令和代码类值使用等宽字体。

禁用模式：

- 营销式 hero 布局。
- 装饰性渐变光斑或虚化背景。
- 绕过 App Shell 的一次性页面风格。
- 普通页面区域中嵌套卡片。
- 依赖 AI 生成装饰图的 UI。

## 8. 可访问性和交互规则

规则：

- 可交互列表行必须有明确的悬停态和选中态。
- 按钮和输入框在浅色、深色主题下都必须保持可读对比度。
- 主按钮文字使用对应主题的 `accentContrast` token，不能假设强调色始终适合白色文字。
- 文本不能溢出容器；路径和命令必须有明确的换行或截断策略。
- 将静态原型迁移为 React 组件时，必须考虑键盘和焦点行为。

## 9. 暂定规则

以下规则已经出现在当前原型中，但应在正式实现时继续确认：

- 最小桌面宽度从 1024px 开始。
- 列表 + 详情页面右侧详情面板宽度约为 396px。
- 项目和 Skills 列表使用表格行，Radar 列表使用 Row Card。
- 侧边栏宽度为 220px。
- Skills 表格行最小高度、全局工具列宽、操作列宽和分类管理弹窗列宽应使用 `design-tokens.json` 中的表格相关 token。

## Known Gaps

以下场景尚未设计：

- 移动端布局。
- 长时间运行的导入进度。
- 可访问性焦点环细节。
- 最终 React 共享组件 API。
