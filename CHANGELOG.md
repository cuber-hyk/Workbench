# Changelog

所有重要变更都会记录在此文件中。

本文档采用 Keep a Changelog 格式，并在适用时遵循语义化版本。

## [Unreleased]

### Added

- 项目、Skills、技能市场、更新页和资源 Radar 列表增加统一分页控件，支持合法范围内页码输入跳转；筛选后回到第一页，翻页时详情自动切换到当前页可见项。
- Skills 支持通过“同步 Skills”统一刷新本地根目录、发现外部工具目录 Skills，并在用户确认后导入和接管工具目录目标。

### Changed

- 外部工具 Skills 同步确认后会在替换工具目录内容前备份原目标，并将目标转为 Workbench 管理的链接或副本；同名不同内容冲突必须选择版本来源。
- “同步 Skills”执行期间使用按钮 loading，只有存在待处理项时打开弹窗；仅发现同名相同内容或无外部候选时改为 toast 汇总。
- 外部工具 Skills 同步弹窗改为按新增、冲突、已存在和不可用分 Tab 处理；冲突批量策略支持选择和取消，确认提示移动到底部摘要区域，候选较多时只滚动当前列表。
- 外部工具 Skills 同步提交时显示“同步中”并禁用重复操作；同步成功后自动关闭弹窗、刷新 Skills 列表并通过 toast 汇总。
- 外部工具 Skills 同步弹窗的空状态、批量策略按钮和重新扫描 loading 调整为更轻量的专业工具样式。

### Fixed

- 修复同步外部工具 Skills 时文件系统扫描可能导致窗口短暂无响应的问题。
- 修复外部同步提交时统一根目录已存在相同内容可能显示为失败或被误接管的问题；现在会自动跳过。

### Security

## [0.2.0] - 2026-06-22

### Added

- Skills 支持发现已注册工具全局目录中的既有 Skills，并可由用户显式导入到当前统一 Skills 根目录。
- 设置页支持切换统一 Skills 根目录后的旧根目录迁移检查、显式迁移和 Workbench 受管启用目标重建。
- 资源 Radar 在同步 GitHub Stars 时检查 GitHub CLI 状态，未配置或未登录时通过右下角提示告知用户运行 `gh auth login`。

### Changed

- ZIP / 文件夹导入遇到同名 Skill 时不再只能跳过，导入结果弹窗允许用户选择冲突项并显式覆盖，覆盖前会备份旧版本。
- 切换统一 Skills 根目录时会先确认，并记录上一个根目录；切换动作本身不会自动迁移内容、重建链接或删除旧目录。
- Toast 通知改为主题适配的卡片样式，支持 neutral、success、warning 和 danger 语义，并提供关闭入口和命令 chip 展示。
- Skills 更新页在没有可检查的 `skills.sh` 来源 Skill 时显示可操作空状态，并将批量确认面板切换为等待可更新项说明。
- 技能市场安装、更新检查和更新执行改为通过隔离临时目录调用官方 `skills.sh` CLI 提取 Skill，再由 Workbench 写入统一 Skills 根目录并维护来源记录。
- 技能市场详情和失败提示更新为 Node.js/npm/npx、网络和 CLI 失败等边界的明确说明，Toast 只保留简短失败通知。

### Fixed

- 修复技能市场安装过程中切换 Skills 子视图或离开 Skills 页面再返回后进度丢失的问题，返回市场页可继续判断安装进度或最终结果。
- 修复自研 GitHub 仓库遍历安装可能处理无关仓库文件并在 Windows 路径上失败的问题。
- 修复点击不可安装市场项时仍请求远程详情并显示 404 错误的问题。
- 修复技能市场安装成功后刷新更新状态失败可能导致页面空白的问题，失败会保留市场页并显示可见提示。
- 统一技能市场行级按钮和“不支持”等状态徽标的尺寸语义，避免不可点击状态呈现为可点击按钮。

### Security

## [0.1.4] - 2026-06-20

### Added

- Skills 模块新增 `本地 Skills / 技能市场 / 更新` 三个子视图，用户可以在同一页面完成本地管理、市场浏览和来源更新。
- 接入 `skills.sh` 技能市场，支持浏览、搜索、按安装状态筛选、查看远程 Skill 详情，并展示全部、已安装、未安装、可更新、不支持和当前结果数量。
- Workbench 可以直接下载并安装 GitHub 来源 Skill 到统一 Skills 根目录，不依赖 `npx skills`，也不要求用户额外配置 CLI 安装目录。
- 市场安装按钮显示阶段性百分比进度，安装期间锁定其他安装操作，完成后自动刷新市场状态和本地 Skills 列表。
- 市场初次加载改为列表与详情骨架屏，统计区域显示紧凑占位，并在加载失败时提供重试入口。
- 新增 `skills.sh` 来源更新页，支持检查全部、单项更新、选中批量更新和更新全部可更新项，并逐项展示更新结果。
- 新增 `skill_sources` 来源记录，保存 Workbench 从市场安装的 Skill 包标识、仓库、仓库内路径、内容 hash 和最近检查时间，为市场状态和更新检查提供依据。
- 市场更新在替换 Skill 前自动备份统一根目录中的旧版本到 `~/.workbench/backups/skills/market/`，批量更新中的单项失败不会中断其他项。

### Changed

- 更新弹窗的更新说明按条目展示，更适合阅读较长的版本说明。
- 技能市场详情区按描述、安装状态、`skills.sh` 包、Skill ID、来源仓库、参考命令、安全提示和 `SKILL.md` 预览分层展示；安装入口统一保留在列表操作列。
- 技能市场列表在当前应用进程内复用已加载数据，切换子视图时不再重复请求；用户仍可通过“刷新市场”强制重新加载，安装或卸载完成后缓存会自动失效。
- 技能市场状态改为语义图标标签，区分未安装、已安装、可更新、不支持和检查失败；“已安装”和“不支持”不再伪装成可点击按钮。
- 技能市场已安装条目的操作列改为“卸载”动作，卸载会删除统一 Skills 根目录中的源 Skill、清理 Workbench 管理的启用副本或符号链接，并移除 `skill_sources` 来源记录。
- 市场安装进度改由后端阶段事件驱动，下载、解压、复制和来源记录写入完成前不再停留在前端模拟的 92%。
- 市场安装和更新后的 Skill 默认只写入统一 Skills 根目录，不自动启用到任何 Agent 工具目录；已启用的 Copy 副本也不会随市场更新自动重同步。
- 市场暂只支持 GitHub 来源 Skill 的 Workbench 自管安装；非 GitHub 来源条目会标记为不支持并禁用安装操作。
- Skills 页标题、子视图 tab 和页面操作在常用窗口宽度下保持同一行；仅在更窄窗口下换行，以增加列表和详情的可用纵向空间。

### Fixed

- 修复 Skills 列表全局工具列点击 `+N` 后浮层被裁切、无法看到更多工具的问题。
- 统一本地 Skills、技能市场和更新页的操作列表头与按钮对齐及列宽，避免默认窗口下“安装”等操作按钮溢出或被裁切。
- 修复本地 Skills 表格操作列仍被图标操作组靠右规则影响的问题；本地 Skills 保持紧凑图标按钮，市场和更新页按文字按钮自适应宽度。
- 修复 `skills.sh` 更新页可更新项复选框无法稳定勾选的问题；更新页复选框仅允许选择可更新项，已是最新、检查失败或更新执行中的条目保持禁用，以确保批量更新只作用于已选可更新项。
- 增大桌面应用默认窗口宽度，降低默认窗口下表格操作列被裁切的概率。
- 清理从 `skills.sh` 页面数据解析出的 Next.js 内部占位标记，避免 `$2b` 等无意义内容出现在 `SKILL.md` 预览中。

### Security

- 市场安装和更新在拼接本地路径前校验 Skill 目录名，拒绝绝对路径、父目录和嵌套路径，防止绕过统一 Skills 根目录写入或替换文件。
- 市场下载增加连接超时、读取超时、100 MiB 压缩包上限、500 MiB 解压上限和仓库文件数量上限，避免异常远端仓库长期阻塞或占用过多本地资源。

## [0.1.3] - 2026-06-18

### Added

- 关闭主窗口默认隐藏到托盘并首次提示；设置页可切换为退出应用，托盘菜单支持显示 Workbench 和退出应用。
- Skills 全局启用扩展支持 DevEco Code、Hermes、Kimi Code、Pi Agent、Gemini CLI、Qwen Code、Goose、Kilo Code、Cline、Roo Code、Factory Droid、Amp、Kiro CLI 和 Junie CLI，并在设置页支持调整全局工具展示顺序。
- Skills 增加按启用工具和启用项目筛选；工具筛选同时覆盖全局启用和项目级启用。
- Skills 增加分类实体表和分类管理弹窗，支持新增、重命名、删除迁移和合并分类。

### Changed

- Skills 列表的全局工具列改为前 4 个彩色工具图标加 `+N` 浮层，项目启用详情区仅展示支持项目级 Skills 的工具。
- Skills 列表隐藏可见滚动条，并收紧表格列宽以减少默认窗口下的横向溢出。
- 设置页打开不存在的工具 Skills 目录时，会先确认是否创建对应目录。
- 整合项目详情面板中的启动配置和本次启动会话摘要，以紧凑启动项列表展示配置、会话状态和单项操作。
- 将 Skills 分类编辑从双击标签改为列表中的分类下拉，并支持直接创建新分类；详情区不再重复展示分类。
- 将 Skills 分类元信息从文本分类迁移为 `category_id`，分类展示名统一来自 `skill_categories`。

### Fixed

- 修正项目表格启动项、状态和操作列的表头对齐。

## [0.1.0] - 2026-06-17

### Added

- 增加应用更新入口：启动后静默检查 GitHub Releases，新版本可在左下角提示并进入设置页确认下载、安装和重启。
- 建立可运行的 Tauri、React 和 TypeScript 桌面应用骨架。
- 实现项目、Skills、AI Radar、设置四个 MVP 页面及列表详情工作区。
- 增加浅色与深色主题、共享 UI 组件和 mock API 边界。
- 增加添加项目、导入 Skills 和添加 Radar 条目的 UI 弹窗流程。
- 实现 Skills 统一根目录扫描、分类、ZIP / 文件夹导入和本地路径打开。
- 实现 Codex、Claude Code、OpenCode 的全局及项目级符号链接启用。
- 使用 `~/.workbench/workbench.sqlite` 保存 Skills 设置、分类和启用关系。
- Skills 启用默认使用 Auto 同步：优先 Symlink，失败时回退 Copy，并记录实际同步方式。
- Skills 导入使用系统选择器选择 ZIP 文件或已解压文件夹，并支持一次导入多个 Skills。
- Skills 扫描可识别全局工具目录中的内容一致启用和内容冲突；内容一致时自动登记为 Workbench 管理，内容冲突时备份后解决。
- 增加删除 Skill 功能，删除时清理 Workbench 管理的启用记录和受管目标。
- 增加项目级 `AGENTS.md`、`CONTEXT.md`、context map、capabilities、audits 和 ADR 文档入口。
- 增加项目启动后端，在 Tauri 桌面应用中通过新的系统终端窗口执行启动命令。
- 增加项目管理 SQLite 持久化，项目新增和编辑会保存到本地数据库。
- 项目启动配置支持多个启动项，启动项目时会执行所有启用启动项。
- 项目路径和启动工作目录支持通过系统目录选择器填充。
- 增加 AI Radar SQLite 持久化与真实增删改查、搜索筛选、收藏和打开链接能力。
- 增加统一 `pnpm verify` 验证入口，并补齐项目多启动项选择和 Skills 目标不覆盖回归测试。
- 增加项目归档和恢复能力，归档只影响 Workbench 本地记录。
- 增加前端交互测试，并将其接入 `pnpm verify`。
- 增加项目内嵌启动会话，支持按启动项查看本次 stdout / stderr 输出并停止运行中的会话。
- 增加项目启动日志详情页，支持在本次启动内按全部或单个启动项查看输出。
- 增加资源 Radar 的 GitHub Stars 手动同步、来源筛选和仓库元数据显示。
- 增加资源 Radar 单选领域、领域筛选、语言筛选、来源状态筛选和重复状态筛选。
- 增加 GitHub Stars 与手动资源的 URL 去重、来源合并和多候选重复组合并流程。
- 增加项目打开方式 Profiles，可从项目列表用 VS Code、Trae、PowerShell、Claude Code 等外部工具打开项目，并在设置中配置 PATH 命令或可执行文件路径。

### Changed

- 将项目页面从卡片列表调整为高密度表格 + 右侧详情布局。
- 将项目打开目录、启动和编辑入口迁移到项目表格操作列，并增加启动配置状态筛选。
- 补齐项目添加和编辑弹窗的提交行为，项目路径填写后自动默认启动工作目录。
- 优化桌面工作区高度布局，减少页面、列表、详情和弹窗之间的重复滚动条。
- 调整 Skills 全局工具冲突处理布局，将版本选择和打开目录等辅助操作分层展示。
- 将 Skills 筛选中的启用状态改为状态筛选，支持快速定位内容冲突。
- 将 Skills 冲突解决调整为 Skill 级唯一版本源选择，避免按工具分别选择导致前后覆盖。
- 将 Skills 列表中的全局工具标识改为可点击开关，并支持双击分类标签直接编辑分类。
- 将删除 Skill 入口迁移到 Skills 表格操作列，并移除详情页危险区。
- 将全局工具启用完全迁移到 Skills 表格，详情页仅在冲突时显示冲突解决面板。
- 将打开 `SKILL.md` 入口迁移到 Skills 表格操作列，详情页仅保留文件路径信息。
- 更新设计 token，固化 Skills 表格图标按钮、行高和列宽规则。
- 将 Skills 表格中的全局工具入口改为参考 cc-switch 的工具图标。
- 为单个项目启用增加项目级总开关。
- 优化 Skills 导入结果弹窗，增加摘要、状态列表、打开统一根目录和更直白的导入边界说明。
- 校准浅色和深色主题对比度，并补齐列表行和筛选控件的可访问语义。
- 优化项目启动反馈、基础空 / 加载 / 错误状态、表单校验和设置页本地路径入口。
- 将项目启动从外部系统终端窗口改为 Workbench 内非交互式启动会话。

- 将 AI Radar 扩展并改名为资源 Radar；GitHub Stars 同步不会覆盖用户分类、标签、备注和收藏，取消 Star 只标记来源失效。

- 收敛 Workbench 跨模块 UI 语义，统一行级操作、状态徽标、详情标题、更多筛选和删除确认样式；资源 Radar 详情保留编辑和打开链接图标，删除迁移到 Row Card 行级操作。

- 增加启动会话内存快照同步，避免日志事件链路异常时日志详情页一直显示等待输出。
